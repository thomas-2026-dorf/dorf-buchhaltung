----------------------------------
DORF-BUCHHALTUNG – PROJEKTSTATUS
----------------------------------

Projekt:
Dorf Buchhaltung (Vereins-/Kassenlösung)

Ziel:
Einfaches, sicheres Buchhaltungssystem für einen Kassierer

Technik:
- Vite + React + TypeScript
- Tauri Desktop App
- Lokale Speicherung (JSON)
- Keine Datenbank (kein Supabase)

===== FUNKTIONSÜBERSICHT =====

Vorhanden:
- Belegverwaltung (aus FeWo übernommen)
- Bankmodul (Import + Zuordnung)
- Exportmodule (Basis vorhanden)

In Vorbereitung:
- Mitgliederverwaltung
- Beitragslogik (Mitgliedsbeiträge)
- Kassenmodul (Bank + Barkasse)
- Rechenschaftsbericht

===== WICHTIGE SYSTEMREGELN =====

- Keine SQL-Datenbank verwenden
- Keine Supabase-Anbindung
- Daten ausschließlich lokal (JSON)

- Änderungen an Bank/Export nur vorsichtig
- Keine automatische Logik ohne Kontrolle
- Bestehende Struktur nicht unnötig umbauen

===== ARBEITSWEISE =====

- Nur kleine, sichere Schritte
- Keine großen Refactorings
- Immer exakte Code-Anweisungen
- Nach jedem Schritt stoppen und testen

===== OFFENE THEMEN =====

- Mitglieder an Bankbuchungen koppeln
- Beiträge automatisch erkennen
- Kassenlogik (Bank vs Barkasse)
- Jahresübertrag korrekt umsetzen
- Rechenschaftsbericht erzeugen
